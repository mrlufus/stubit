The Idea

// What is a DNS and how will we use it?

The name DNS stands for Domain Name System and is widely used to resolve hostnames like "google.com".
So if you open a new Browser Window and type in "google.com" a DNS request will be sent to your predefinded DNS Server like this:


   (You) Browser: DNS request for "google.com" to (example) DNS Server IP(1.1.1.1)
         Server: The IP Address for the Domain "google.com" is IP(216.58.207.46)

The Browser asked for the IP Address of "google.com" and the DNS Server returned the desired IP Address "216.58.207.46".
After this request your browser opens the corresponding website and you can search the web!


As you saw, a working DNS Server is essential to surf the internet. And with this in mind we can block access to unwanted sites with a simple trick.
Just Open a new Tab and type in "0.0.0.0". Your browser will tell your something like that: "This Site can't be reached - 0.0.0.0 refused to connect".

So but what happens here? - You simply entered an IP Address instead of an Domain Name. And "0.0.0.0" is not routable, meaning no server responds to it.
With this in mind we can create a new DNS Server which doesn't respond to queries like "instagram.com" and so returns "0.0.0.0".
